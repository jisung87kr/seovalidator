## Technical Details
### Implementation Approach
- Laravel Queue with Redis driver
- 작업 유형별 큐 분리 (high, normal, low priority)
- Job 체이닝으로 복잡한 워크플로우 처리
- 실패한 작업 관리 및 알림 시스템

### Queue Structure
```php
// Main Analysis Workflow
AnalyzeUrlJob::dispatch($url)
  -> CrawlPageJob
  -> TechnicalSeoAnalysisJob
  -> ContentSeoAnalysisJob
  -> CalculateScoreJob
  -> NotifyCompletionJob

// Queue Configuration
- 'high': API 요청, 실시간 분석
- 'normal': 일반 분석 작업
- 'low': 리포트 생성, 데이터 정리
```

### Key Jobs
```php
// AnalyzeUrlJob
- handle(): 전체 분석 워크플로우 조정
- failed(): 실패 시 처리 로직

// CrawlPageJob
- handle(): 페이지 크롤링 및 기본 데이터 추출
- timeout: 60초

// GenerateReportJob
- handle(): PDF/Excel 리포트 생성
- queue: 'low' priority

// CleanupOldAnalysisJob
- handle(): 30일 이상 된 분석 결과 정리
- schedule: 일일 실행
```

### Code Locations
- `app/Jobs/` - 모든 Job 클래스들
- `config/queue.php` - 큐 설정
- `database/migrations/` - 큐 관련 테이블
- `app/Console/Commands/` - 큐 관리 명령어
